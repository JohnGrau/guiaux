Guía de Experiencia de Usuario

Objetivos

Este documento especifica la línea estética, de usabilidad y de accesibilidad que deberán respetar todos los nuevos desarrollos web y de sistemas de gestión basados en web del Gobierno de la Ciudad de Buenos Aires. Los objetivos son:

  • Los sitios web y sistemas de gestión basados en web deben tener una concordancia estética homogénea que se corresponda con el plan gráfico integral del Gobierno de la Ciudad.
  • Se debe garantizar que los sitios web y sistemas de gestión basados en web puedan ser usados por la mayor cantidad de usuarios, incluyendo usuarios con limitaciones motrices y tecnológicas.

Este documento se actualiza periódicamente. Al momento de inicio de cada proyecto se deberá usar la última versión de los lineamientos que esté disponible.

BAStrap / El framework

Todo nuevo desarrollo de:

  • Sitio web
  • Sitio web para plataformas móviles
  • Landing page
  • Aplicativo interno de Gobierno
  • Aplicativo externo de Gobierno

deberá usar el framework basado en Bootstrap llamado BAstrap: http://gcba.github.io/BAstrap/

En el caso de no usar este framework, consensuado con la DG de Gobierno Electrónico, se deberán aplicar los lineamientos de la sección Proyectos sin BAStrap.

Beneficios de uso de un framework

El uso de un framework para todos los sitios del Gobierno tiene muchos beneficios:

  • Unificar línea estética
    La estética del Gobierno de la Ciudad debe ser homogénea. A través de un framework se puede seguir una misma línea estética sin requerir grandes trabajos de diseño.
  • Garantizar un nivel mínimo de usabilidad y accesibilidad
    El framework BAStrap cuenta con muchas características de accesibilidad y usabilidad incorporadas en su código, como avisos de error para formularios, funcionamiento instantáneo en dispositivos móviles, navegación por teclado.
  • Optimizar los tiempos de desarrollo
    Al no tener que pensar en cuestiones elementales del diseño, maquetado, y funcionalidades, se pueden concentrar los esfuerzos en el desarrollo.
  • Reutilización de componentes y universalidad de recursos
    Todos los componentes que se creen para una plataforma de un sitio de gobierno estarán disponibles para los otros sitios web.
  • Mantenimiento universal y actualización simultánea
    Al centralizar los recursos, las actualizaciones y mejoras a cualquier parte del código fuente impactarán automáticamente en todos los desarrollos.

Las áreas podrán desarrollar módulos o estilos sobre el base de BAstrap. Estos desarrollos deberán ser supervisados y aprobados por la DG de Gobierno Electrónico. Los desarrollos deberán cargarse como ampliaciones a BAstrap y su código se liberará para poder ser reutilizado por otras áreas del gobierno.

Proyectos sin BAStrap

Los proyectos que no usen BAStrap deberán aplicar la marca del Gobierno de la Ciudad en las siguientes maneras:

Aplicación de marca

Encabezado

Se deberá aplicar al inicio de todas las páginas una barra que contenga el escudo de la Ciudad y el lema “En todo estás vos” tal como aparece en esta misma página.

Pie

Se deberá aplicar al final de todas las páginas una barra que contenga el escudo de la Ciudad enlazando a www.buenosaires.gob.ar tal como aparece en esta misma página.

Uso de sellos del Gobierno

Se aplicarán los sellos “Ciudad Verde”, “Agenda Cultural” o “Red En Todo Estás Vos” en los casos en que el sistema creado esté alineado en alguno de los tres ejes.

Otros

  • El tamaño mínimo de la tipografía será de 12px para cuerpo de texto y de 10px para anotaciones secundarias.
  • El uso de íconos deberá estar acordado con la DG de Gobierno Electrónico, que guarda un listado de íconos oficial.

Tono

El tono de comunicación es joven e informal, lo que implica hablar de igual a igual, no ser imperativo, ser auténtico, cercano y directo.

Se centra la atención sobre los temas importantes. Se elimina la comunicación superflua.

Los mensajes deben ser principalmente positivos, integradores e inclusivos.

La excepción: El tono informativo. El tono informativo y formal se reservará únicamente para comunicaciones de temas muy precisos que requieran extrema claridad en transmitir el mensaje.

Es requisito que las aplicaciones web del Gobierno de la Ciudad se desarrollen respetando los siguientes requerimientos mínimos de accesibilidad:

Requerimientos mínimos de accesibilidad

  • Proveer texto alternativo en todas las imágenes esenciales. Las imágenes que son solo decorativas no lo requieren.
  • Utilizar subtítulos en video
  • Contraste mínimo del texto 4.5:1
    Para un fondo blanco #FFF, el texto debería ser #888 o más oscuro.
  • El tamaño del texto se debe poder ajustar desde el navegador
  • Las páginas deberán funcionar correctamente en escalas de grises
  • Las columnas de texto no deben ser mayores de 110 caracteres
  • Texto no justificado
  • Interlineado mínimo de 1,5
  • Espacio entre párrafos mínimo 150% del interlineado
  • Los componentes que pueden recibir el foco lo hacen en un orden que preserva su significado y operabilidad
    Estos componentes generalmente son campos de un formulario y enlaces. El teclado deberá poder navegarlos correctamente.
  • El usuario puede continuar la actividad sin pérdida de datos cuando expire una sesión autenticada
  • Las páginas web no contienen nada que destelle más de tres veces en un segundo
  • Los títulos de las páginas deben describir su temática o propósito
  • El idioma predeterminado de cada página web será explicitado en el código HTML
